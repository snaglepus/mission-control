<task type="auto">
  <name>Task 2: Add saturate(140%) to glass-card backdrop-filter</name>
  <files>src/app/globals.css</files>
  <action>
**In src/app/globals.css** `@layer components`, update the `.glass-card` class to add `saturate(140%)` alongside the existing `blur(12px)`:

Change:
```css
backdrop-filter: blur(12px);
-webkit-backdrop-filter: blur(12px);
```

To:
```css
backdrop-filter: blur(12px) saturate(140%);
-webkit-backdrop-filter: blur(12px) saturate(140%);
```

Keep BOTH the unprefixed and `-webkit-` prefixed declarations — Safari still needs the prefixed version per caniuse.com 2025 data.

**Why 140%:** Values above 100% increase color saturation of the backdrop. At 140%, the warm background colors bleed through glass cards more richly. Values above ~180% look artificially oversaturated.

**Also update .glass-card background and border** to use the warm token values while you're in this block:

Change the `.glass-card` background from `rgba(15, 23, 42, 0.6)` to `var(--card)` and border from `rgba(99, 102, 241, 0.15)` to `var(--card-border)` so it picks up the warm values defined in Task 1.

Change `.glass-card:hover` border-color from `rgba(99, 102, 241, 0.3)` to `rgba(251, 191, 36, 0.3)` (warmer hover) and box-shadow from `rgba(99, 102, 241, 0.1)` to `rgba(245, 166, 35, 0.1)` (amber glow).

Do NOT change the `.neon-text`, `.neon-border`, `.gradient-text`, `.sidebar-item`, or `.card-gradient` classes — those are Phase 2-5 work.
